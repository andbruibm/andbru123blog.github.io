使用Gridea在github pages上搭建静态博客

2023-09-03 2 min read

如果您计划创建自己的博客站点，并寻找一种简单而强大的方式来管理内容，那么Gridea可能是一个不错的选择。在本文中，我们将向您介绍如何在 GitHub Pages 上使用 Gridea 搭建静态博客站点的步骤。

下载和安装 Gridea

首先，前往Gridea 官网您会在页面下方找到一个下载图标点击它，您将被引导到 Gridea 的 GitHub 仓库页面。如果您更倾向于使用国内 Gitee 下载源，可以访问此链接 获取最新版本。

  1. 下载 Gridea 后，双击运行安装程序。如果您不想将软件默认安装在 C 盘，可以自定义安装目录

  2. 安装完成后，运行软件。在主界面中，点击 "远程" 您将看到多个平台供您选择，这里请选择 "GitHub Pages"。

配置 GitHub 仓库

在配置 Gridea 与 GitHub Pages 之前，确保您已经创建了一个 GitHub 仓库来托管您的博客内容，并启用了 GitHub Pages 设置。以下是配置步骤：

  • 填写域名和仓库名称，确保与您的 GitHub 仓库设置匹配。
  • 分类可以默认选择 "master"。
  • 仓库用户名应该是您的 GitHub 用户名。
  • 邮箱字段可以留空。
  • 令牌需要通过 GitHub 的令牌生成页面 获取，确保它具有必要的权限。
  • 如果您计划使用自定义域名，请填写 CNAME 字段。

完成上述步骤后，您已经成功配置了 Gridea 以在 GitHub Pages 上搭建静态博客站点。现在您可以开始编写和发布博文，将您的想法和内容与世界分享！
